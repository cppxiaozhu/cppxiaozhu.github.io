生效作用域

默认状态下，const 对象仅在文件内有效，多个文件出现同名的 const常量 时，等同在不同文件下分别定义了独立的常量，若想它在多个文件中共享可以这么做：

const 引用

  • 非常量引用，不可引用常量，否则就会造成常量被修改的逻辑谬误；

  • 初始化常量引用时，允许任意方式赋初值；

  • 常量引用，可以引用一个非常量，毕竟人家只是取个地址；

  • 常量引用某非常量后，可以通过另一个普通引用修改该非常量；

常量引用非常量，仅仅是限制自己参与修改对象值的行为，不管别人，甚至不管被引用对象变没变，毕竟，人家只是取个内存地址；

const 指针

指向常量的指针

形如 const int *curErr ，阅读顺序「从右向左」，指针指向了常量，这代表着指针自以为是地认为自己指向了常量，并自觉地不去改变它所指对象的值，但是它也可以指向一个非常量。其实，该针指仅仅约束了自己被解引用改对象值的操作，但它可以改嫁，指向别人。

常量指针

指针是「对象」，而「引用」不是。因此，允许把「指针」本身定为「常量」。* 号在 const 之前，就说明：该指针是常量，称为 常量指针 。
阅读顺序「从右向左」，形如 int *const curErr 即是常量指针。

指针被修饰，地址永不变

顶层 const

  • 若数据类型为指针，则指针本身是常量，即为「顶层const」属性；

其实就是常量指针：int *const curErr

顶层 const 指针保持所指对象的地址不变，至于后续对象的值变不变，它不管。

底层 const

  • 若数据类型为指针，则指针所指的对象是常量，即为「底层const」属性；

const int i = 13;int errNumb =0;const int *curErr= &i;

非常量指针，是不能指向常量的，既然已是「指向常量」的「指针」，「指针」本身也必是常量。
